# SEXTANT
## Simple Microsimulation Engine

**S.E.X.T.A.N.T. = Synthetic Evolution eXtension for Temporal Attribute Navigation in Time**

Sextant is a lightweight, simple microsimulation engine designed to add a **temporal component** to synthetic populations. It applies basic, rule-based changes to population attributes over time—aging individuals, applying mortality, and simulating births.

**Sextant is a simple, limited version of the full Hermes microsimulation engine.** It provides the core functionality needed to evolve static synthetic populations forward in time, without the complexity of a full-scale microsimulation system.

---

## What Sextant Does

Sextant takes a static synthetic population (generated by tools like **COMPASS**) and evolves it forward in time. Each year, it applies a set of user-defined transition models to each individual, updating their characteristics.

**In short:** Sextant gives your synthetic population a history.

**Key limitations:**
- Simple attribute updates only (age, income, mortality, fertility)
- No complex interactions between individuals
- No household formation or migration
- No spatial dynamics beyond basic area grouping

For more complex microsimulation needs, consider the full **Hermes** engine.

---

## How It Works

Sextant operates on three simple components:

| Component | Description |
|-----------|-------------|
| **CSV Population** | Your population data. Each row is one person, each column is an attribute (age, income, health status, etc.). |
| **YAML Configuration** | Defines which models to run, in what order, and with what parameters. |
| **Lua Models** | Simple scripts that describe how attributes change over time (e.g., "age increases by 1 each year"). |

The engine reads your population, applies the models year by year, and outputs the updated population.

---

## Example: Aging a Population

A simple aging model in Lua:

```lua
function transition(population, params)
  for _, person in ipairs(population) do
    if person.alive == true then
      person.age = person.age + 1
    end
  end
  return population
end
```

## Quick Start

### 1. Download Sextant

Pre-built binaries are available for:

| Platform | Binary |
|----------|--------|
| Linux (x86_64) | `sextant-linux-amd64` |
| Linux (ARM64) | `sextant-linux-arm64` |
| Windows | `sextant-windows-amd64.exe` |
| macOS (Intel) | `sextant-darwin-amd64` |
| macOS (Apple Silicon) | `sextant-darwin-arm64` |

### 2. Create a Configuration

```yaml
simulation:
  iterations: 5
  population_file: "population.csv"
  output_file: "output.csv"
  id_column: "person_id"
  streaming_mode: false

models:
  - name: "age_increment"
    type: "lua_model"
    priority: 1
    enabled: true
    parameters:
      script: |
        function transition(population, params)
          for _, person in ipairs(population) do
            if person.alive == true then
              person.age = person.age + 1
            end
          end
          return population
        end
```

### 3. Run the Simulation

```bash
./sextant-linux-amd64 config.yaml
```

## Configuration

### Simulation Parameters

```yaml
simulation:
  iterations: 10                    # Number of years to simulate
  population_file: "population.csv" # Input CSV file
  output_file: "output.csv"         # Output CSV file
  random_seed: 42                   # Fixed random seed for reproducibility
  verbose: true                     # Detailed logging output
  id_column: "person_id"            # REQUIRED: Unique identifier column
  streaming_mode: false             # Set to true for large populations
```

### Model Definitions

```yaml
models:
  - name: "model_name"
    type: "lua_model"
    priority: 1
    enabled: true
    description: "What this model does"
    parameters:
      script: |
        function transition(population, params)
          for _, person in ipairs(population) do
            -- Your logic here
          end
          return population
        end
```

## Input Data Format

Sextant accepts any CSV file with a header row. Column types are automatically detected.

**Required columns:**
- `person_id`: Unique identifier (must match `id_column` in config)
- `age`: Age in years
- `alive`: Boolean (true/false)

**Recommended columns:**
- `area`: Geographic area
- `sex`: Gender

### Example CSV

```csv
person_id,age,sex,area,alive
1,25,F,north,true
2,30,M,south,true
3,45,F,east,true
```

## Streaming Mode (for Large Populations)

For populations of 1 million or more, enable streaming mode:

```yaml
simulation:
  streaming_mode: true
  area_column: "area"   # CSV must be sorted by this column
```

Streaming mode processes the population area-by-area, keeping memory usage constant (~20-50MB) regardless of population size.

---

## Performance

| Mode | Memory Usage | Best For |
|------|--------------|----------|
| **Bulk** | ~2MB per 1000 people | Populations < 1M |
| **Streaming** | ~20-50MB constant | Populations 1M+ |
